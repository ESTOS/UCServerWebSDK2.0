asnMeshTwoWayAuthenticatePart1

Two way authentication process part1
called from server in client role

  1. we generate a new random seed A
  2. we SHA-1 hash the concatenation of our password and seed A
  3. we send the seed A to the remote server
  4. remote server generates a new random seed B
  5. remote server sends back SHA-1 hash of concatenation of its password and seed A, and meshid, and seed B
  6. we test remote meshid with our meshid and remote hash with our hash
  7. if successfully tested we call asnMeshTwoWayAuthenticatePart2
