asnACUCheckUpdateNow

Ask the server if we are allowed to install the downloaded update file now

If one of the clients running on the same machine as the update service
has an active AV call, the server will reject the installation of the update.
The server evaluates the status of all clients on the machine and check that no av call is active.
If the update server is configured to force the update the server will allow the update no matter if av calls are active or not.
If the update server is configured to wait for all media calls beeing terminated the client has to implement the business logic
If the result denies an update the client has to subsequent ask for permission.
Depending on the iMaxWaitTimeInMinutes the client may process the update after the mentioned period even if media calls are active.

ArgumentType: AsnACUCheckUpdateNowArgument

- PRIVATE -

AsnACUCheckUpdateNowArgument is an Object

Name Type Optional Comment

u8sClientComputerName

string

Computer Name of the client

ResultType: AsnACUCheckUpdateNowResult

- PRIVATE -

AsnACUCheckUpdateNowResult is an Object

Name Type Optional Comment

bUpdateNow

boolean

true if the client is allowed to install the update now, false if not

iMaxWaitTimeInMinutes

number

If specified, this is the configured maximum Wait time the client has to wait until it forces the update.
If this value is 0 the client must wait until bUpdateNow gets true
If this value is greater than 0 the client must store the timestamp it first received it.
